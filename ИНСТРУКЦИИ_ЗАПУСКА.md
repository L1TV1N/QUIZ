# 🚀 Smart Quiz - Инструкции по запуску

## ✅ ЧТО БЫЛО СДЕЛАНО В ЭТОЙ ИТЕРАЦИИ

### Frontend (React + Zustand + Framer Motion)
1. **Обновлен quizStore** с новыми полями по ТЗ:
   - `aiMode` - выбор режима (true = AI, false = обычный)
   - `zones` - массив выбранных зон (множественный выбор)
   - `area` - число (не строка) для площади
   - `style` - одиночный выбор (не массив)
   - `phone`, `email`, `comment` - отдельные поля контактов
   - `agreeToTerms` - чекбокс согласия

2. **Добавлены 4 новых шага**:
   - `Step0ModeSelection` - выбор режима (обычный/AI)
   - `Step3Zones` - множественный выбор зон (10 вариантов)
   - `Step4Area` - ползунок площади (20-300 м²)
   - `Step7AiQuestions` - уточняющие вопросы от AI

3. **Обновлены существующие шаги**:
   - `Step2Property` - 6 вариантов по ТЗ
   - `Step4Style` - одиночный выбор (из 7 стилей)
   - `Step5Budget` - только бюджет (5 вариантов)
   - `Step6Contacts` - расширет с phone, email, comment, чекбокс
   - `ResultScreen` - показывает все данные + режим

4. **Переделан App.jsx**:
   - Новая структура с 9 шагами вместо 6
   - Логика переключения между режимами
   - Условное отображение Step7AiQuestions если aiMode = true

### Backend (Flask + SQLAlchemy)
1. **Обновлена модель Lead** в `models.py`:
   - `ai_mode` (boolean)
   - `zones` (text/JSON)
   - `area` (integer)
   - `style` (string)
   - `budget` (string)
   - `phone` (string) - отдельно
   - `email`, `comment` (optiional)
   - `agree_to_terms` (boolean)

2. **Обновлен endpoint** /api/submit-quiz:
   - Принимает новую структуру данных
   - Правильно обрабатывает zones как JSON
   - Поддержка AI-режима

## 📋 НОВАЯ СТРУКТУРА КВИЗА (9 шагов)

```
Step 0: Выбор режима (обычный / AI) ← NEW
Step 1: Введение
Step 2: Тип помещения (6 вариантов)
Step 3: Зоны (множественный выбор) ← NEW
Step 4: Площадь (ползунок) ← NEW
Step 5: Стиль (одиночный выбор)
Step 6: Бюджет (одиночный выбор)
Step 7: AI-вопросы (только если aiMode = true) ← NEW
Step 8/7: Контакты + финальная отправка

Финальный: ResultScreen
```

## 🔧 КАК ЗАПУСТИТЬ

### Требования
- Python 3.8+
- Node.js 14+
- npm или yarn
- pip для Python

### Установка и запуск Backend

```bash
# Перейти в папку backend
cd backend

# Установить зависимости
pip install -r requirements.txt

# (Опционально) Создать .env файл с конфигом
# TELEGRAM_BOT_TOKEN=xxxxx
# GIGACHAT_AUTH_KEY=xxxxx

# Запустить сервер
python app.py

# Проверить: http://localhost:5000/api/health
```

### Установка и запуск Frontend

```bash
# Перейти в папку frontend
cd frontend

# Установить зависимости
npm install

# Запустить development сервер
npm run dev

# Откроется: http://localhost:5173
```

## 🐛 ВОЗМОЖНЫЕ ОШИБКИ И РЕШЕНИЯ

### "Step3Params не найден"
- Step3Params больше не используется в App.jsx
- Файл можно удалить или оставить как архивированный

### "updateZones не существует"
- Проверить, что quizStore импортирует правильного setQuizState
- Функция updateZones все еще там для обратной совместимости

### Frontend не грузится
- Проверить, что npm install выполнен
- Очистить кэш: `rm -rf node_modules && npm install`
- Проверить порт 5173 не занят

### Backend ошибка при импорте models
- Убедиться, что database.db не заблокирован
- Пересоздать БД если нужно: `rm backend/database.db`
